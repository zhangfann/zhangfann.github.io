# 名字节点

名字节点主要存放元数据相关数据, 元数据主要记录两类关系, 分别为第一关系和第二关系,

第一关系记录文件与数据块列表的对应关系,

第二关系记录数据块与节点信息的对应关系

# 数据节点

数据节点是存放具体数据的设备, 数据节点中每个数据块中会保存数据和校验和, 这样数据节点可以自己进行校验, 检查数据是否损坏,

# 注册

在注册时, 上报本节点的数据块信息,

名字节点根据数据块信息, 构建{数据块:节点信息}

# 心跳

每次心跳, 名字节点向数据节点发送无效节点信息列表,

数据节点进行删除

# 读流程

客户端从名字节点获取存放一个数据块的数据节点,

客户端从数据节点读取数据

如果数据节点不在线, 读取副本

客户端获取下一个数据块的数据节点

# 写流程

客户端向名字节点申请一个数据块,

客户端向名字节点分配的数据节点写入数据

如果数据节点无法连接

客户端放弃该数据块, 重新向名字节点申请数据块

客户端向名字节点分配的数据节点写入数据

客户端向名字节点申请下一个数据块,

# 并发写, 租约

支持并发读, 不支持并发写,

通过租约控制并发写

# 小文件

通过将多个小文件id, 指向同一个数据块实现,

第一关系: 小文件文件id -> (数据块id, 数据块内部偏移, 小文件长度),

第二关系: 数据块 -> 数据节点

# 删除

客户端向名字节点删除文件

名字节点将文件相关数据块标记为删除

数据节点连接名字节点上报心跳时,

名字节点向数据节点下发删除

# 数据恢复

  • 名字节点检测到副本不够,

通知一个拥有该数据块的数据节点, 复制到其他数据节点

# 数据节点挂掉

如果数据节点在客户端写入后, 数据节点向名字节点汇报前挂掉, 名字节点感知到错误

假如数据块存在于数据节点1, 数据节点2, 数据节点3

数据节点1挂掉,

则数据节点2, 数据节点3汇报时, 名字节点会指定数据节点2复制到一个正常节点

通知一个拥有该数据块的数据节点, 复制到其他数据节点

如果数据节点在客户端写入时挂掉, 客户端感知到错误:

会选择一个主数据节点, 然后让主数据节点对其他数据节点进行数据复制

通知一个拥有该数据块的数据节点, 复制到其他数据节点

之后数据节点会返回客户端一个恢复成功数据块的位置, 和数据节点列表

客户端可以在这个位置, 继续写入

# 客户端挂掉

参考租约恢复

# 名字节点挂掉,

客户端在写入时, 名字节点挂掉

如果名字节点在给客户端分配完数据节点后挂掉,

此时, 客户端把该数据块写完后, 无法关闭文件, 需人工介入,

数据节点这里, 只是暂时块汇报无法响应

# 租约恢复

软超时, 是客户端每隔2s进行续约

硬超时, 是如果客户端断开, 且没有其他客户端抢夺这个租约造成的超时,

一般是1h

如果客户端刚打开文件就挂了, 还没写数据:

直接关闭文件

abandonBlock调用场景:

当客户端请求一个数据块后, 名字节点返回一个数据节点,

当客户端无法连接这个数据节点后, 就会调用abandonBlock,

让名字节点重新分配一个数据节点

如果客户端在调用abandonBlock后挂掉:

主数据节点是倒数第二个数据块的数据节点(倒数第一个被abandon了)

指定一个主数据节点, 然后主数据节点与其他数据节点计算出一个最短公共长度,

以此最短公共长度作为有效数据,

主数据节点向名字节点申请一个新的版本号, 并同步给其他数据节点, 将这个版本号赋给数据块

# 数据均衡

当有新数据节点上线, 下线, 或者执行副本复制后, 导致节点分布不均匀时,

名字节点会计算每台平均数据块数,

将数据块超过平均值的节点的数据块, 移动到不足平均值的数据节点上
