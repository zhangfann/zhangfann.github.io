    # libevent网络模型

    主要是主线程把连接分派给工作线程

    memcached.c main 处理命令行参数 _单个字符，表示选项， _单个字符后接一个冒号：表示该选项后必须跟一个参数。参数紧跟在选项后或者以空格隔开。该参数的指针赋给optarg。 _单个字符后跟两个冒号，表示该选项后必须跟一个参数。参数必须紧跟在选项后不能以空格隔开。该参数的指针赋给optarg。（这个特性是GNU的扩张） 内部容器数据结构初始化 @assoc_init @assoc.c 内部容器就是memcached中用于存储数据的结构 空闲连接数组初始化 @conn_init 这个是管理和用户的连接

        主线程server启动 @server_sockets

    非常重要, 创建4个工作线程 >> @thread_init @thread.c
    进入主线程事件循环 @event_base_loop

conn_new
    $thread_libevent_process调用
        主线程建立连接是监听socket的建立
        工作线程是监听socket读写
        @event_set(event_handler)
    $server_sockets
        为主线程监听连接事件 
        >> drive_machine.conn_listening
event_handler
    @drive_machine memcached大多数网络事件都交给了drive_machine
drive_machine
    c->state
        $主线程
            _conn_listening 正在连接 还没有accept
                只有主线程会进入
                accept 接受连接
                将socket交给工作线程处理 @dispatch_conn_new
                    向工作线程队列放入任务
                    >> conn_new_cmd

        $工作线程
            _初始阶段
                _conn_new_cmd
                    _已经读入一些数据(c->rbytes > 0) >> conn_parse_cmd
                    _未读入数据 >> conn_waiting
                _conn_waiting 等待新的命令请求
                    >> conn_read
            _读取到数据
                _conn_read 读取数据
                    _未读入数据 >> conn_waiting
                    _读取数据成功 >> conn_parse_cmd
                    _读取错误 >> conn_closing
            
            _conn_parse_cmd 解析命令
                \try_read_cmd 尝试解析命令
                    _二进制协议 @dispatch_bin_command
                    _字符协议 @process_command
                _解析完成
                    ??不太清楚转换
                _为解析完成
                    >> conn_waiting 等待读入更多数据
            _conn_nread ??猜测是根据读入的n个字符执行后调用

    thread.c thread_init 创建工作线程 创建每个线程的libevent的event_base @setup_thread 创建worker线程 @create_worker >> worker线程 @worker_libevent setup_thread 创建线程的event_base 每个libevent的线程都应该有一个自己独立的event_base @event_init 对于读事件的监听 @event_set(@thread_libevent_process 使用thread_libevent_process进行回调) 添加事件操作 初始化工作队列

    worker_libevent worker线程入口
    进入事件循环 @event_base_loop
thread_libevent_process 有读数据时回调
    从cq工作线程队列中取出一个item 
    并建立连接 >> @conn_new(thread_libevent_process调用)
