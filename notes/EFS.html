# 名字节点

元数据

第一关系: map{文件: 数据块列表}, 类似mds上的元数据

第二关系: map{数据块: 节点信息}, 类似cs上的信息

# 数据节点

数据节点中每个block中会保存数据和校验和,

这样dn可以自己进行校验, 检查数据是否损坏,

# 系统设计

# 读流程

客户端从mds获取一个blk的dn

客户端从dn读取数据

如果dn不在线, 读取副本

客户端获取下一个blk的dn

# 写流程

客户端向mds申请一个blk,

客户端向mds分配的dn写入数据

如果dn无法连接

客户端abandonBlock, 重新向mds申请blk

客户端向mds分配的dn写入数据

客户端向mds申请一个blk,

# 删除

客户端向mds删除文件

mds将文件相关blk标记为删除

dn心跳连接mds,

mds下发删除

# 数据恢复

1 mds检测到副本不够 通知一个拥有该blk的dn, 复制到其他dn 2 sdk, dn全量汇报检测数据出错 通知一个拥有该blk的dn, 复制到其他dn

# dn挂掉

如果dn在sdk写入后, dn向mds汇报前挂掉, mds感知到错误

假如blk存在于dn1, dn2, dn3

dn1挂掉,

则dn2, dn3汇报时, mds会指定dn2复制到一个正常节点

通知一个拥有该blk的dn, 复制到其他dn

如果dn在sdk写入时挂掉, sdk感知到错误:

会选择一个主dn, 然后让主dn对其他dn进行数据复制

通知一个拥有该blk的dn, 复制到其他dn

之后dn会返回sdk一个恢复成功数据块的位置, 和dn列表

sdk可以在这个位置, 继续写入

# sdk挂掉

参考租约恢复

# mds挂掉,

sdk在写入时, mds挂掉

mds只能在给sdk分配完dn后挂掉, 如果之前挂, 则sdk无法找到写的dn

此时, sdk把该blk写完后, 无法关闭文件, 需人工介入,

dn这里无所谓, 只是暂时块汇报无法响应

# 高可用, 备机做fsimage

备机从主机拉取editlog和image合成新的image,
再将新的image推送给主机

# 注册

在注册时, 上报本节点的数据块信息, 名字节点根据数据块信息, 构建{数据块:节点信息}

# 心跳

每次心跳, 名字节点向数据节点发送无效节点信息列表, 数据节点进行删除,

# 并发, 租约

# 租约恢复

软超时, 是sdk每隔2s进行续约

硬超时, 是如果sdk断开, 且没有其他sdk抢夺这个租约造成的超时,

一般是1h

如果sdk刚打开文件就挂了, 还没写数据:

直接关闭文件

abandonBlock调用场景:

当sdk请求一个blk后, mds返回一个dn,

当sdk无法连接这个dn后, 就会调用abandonBlock,

让mds重新分配一个dn

如果sdk在调用abandonBlock后挂掉:

主dn是倒数第二个blk的dn(倒数第一个被abandon了)

指定一个主dn, 然后主dn与其他dn计算出一个最短公共长度,

以此最短公共长度作为有效数据,

主dn向mds申请一个新的版本号, 并同步给其他dn, 将这个版本号赋给blk
