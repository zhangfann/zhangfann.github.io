有个现象，你不得不说，那些业务经验丰富，参与过企业重要项目的系统架构 (opens new window)，带过团队，或者负责过某项目研发，并产生盈利，且在某一语言能够达到精通水准的老家伙们，是吃香的，是受欢迎的，是巅峰的。

初级到中高级程序员，从你学会CCCV到用自己的逻辑写代码，独立负责核心模块开发，独立负责区域版本开发，组织规划一个项目的开发和交付、维护……这是一条技术进阶的道路，与此同时，还有各种分叉线，比如多门语言，各种数据库等等。

这些，抛除天赋异禀的，要从熟悉到掌握，至少你得经历一遍吧，抛除老板费尽心思给你接活，哪怕是抄一遍的经历总要有吧。然后，三到五年就这么过去了。这你还得保证不是几个月换一家，哪个企业开发一个项目最少也得两个月吧。

更何况，你试用期还得三个月呢。

熬过三五年，开始迷茫，人快到中年，秃头，肥胖，无妻，钱还没跳跳乐多。自己又开始啥都会一点，啥都不精通的迷糊状态。这时候，是开始跳跳乐去一家财富自由呢，还是和老东家死磕到底。想想，老东家虽然抠门，但实践的多啊，什么玩意都敢真刀真枪让你干，干垮了，他顶头，干不垮，夸你一声让你接着干。就是钱不舍得给，抠抠搜搜。

行吧，苦熬着，开始做核心，带团队，反正小猫三两只，新人如流水，谁也不知道手下为啥有一天哭着崩溃了，反正你都麻木了。带团队技术菜怎么行，总不能问小弟这个接口该怎么写。只能自己学，学大佬，学视频，揪着头发做，没搞过的新架构，没搞过的新插件，没搞过的新需求，薅光头发，熬夜加班。没办法，又不是天赋异禀，只能傻干。就好像，一个六角形 (opens new window)的孔，天赋异禀的会测量好，找到诀窍，笨的就只能搓一刀量一下。

怎么着八九十，十年就过了吧。二十二三岁大学毕业，没干劲考研，没本事考公务员，一年半载再学了一门技术，从学徒到老手，从老手到师傅苦熬奋战已十年，三十五了，啥也搞过了，啥也见过了，新的旧的，反正哪怕没学过，总有学习的经验，上手也快。好吧，笨一点的，技术没多精，好歹老资历带过人，好歹谈过几场客户，好歹处理过几次上线交付，好歹写过各种文档说明书，算是一技之长傍身，也算吃过猪肉。

这种一般不担心自己三十五是不是低谷，哪怕他自己出去打天下，就算出去卖煎饼果子，总是能比刚出门的计划的周全，也更有胆子吆喝吧，再不济要是做人不差，好歹有几个玩的好的同事朋友捧捧场吧，帮你推广宣传吧。何况，很多发展中企业非常需求这种领导者，技术扎实能带小弟，本领过硬能扛担子。

（不要说什么小企业没有研发部，我刚出道那会去应聘，就有一家做衣服的厂子准备组建自己的研发中心，和他们负责人相谈甚欢，拿到offer，承诺一年半载给个技术经理，可惜当时技术太差，实在没胆子去给人扛把子，所以面试吹牛千万别吹太厉害，我当时就自我营造成那种看过大世面，组织能力强，行动有计划的样子。实际就是战五渣，水平基本上在刚看完入门手册那种状态，耐不住天生嘴皮子利索，而且还不要脸。免费赠送：面试三要素：胆大心黑不要脸，胆大，有啥说啥不要畏畏缩缩，心黑，工资福利要略高于对方的预期，让对方觉得努力一下就能得到你，当然事先要做好功课。不要脸，七分要说成九分，十分要说成二十分。你干过的项目，你要自我认为自己绝对是主要负责人，和你一起干的不是菜鸡就是笨蛋，你永远都在指导他们，永远是你力挽狂澜，是你百折不挠，是你一波三折，要比小说还精彩。。。。）

那么，那些所谓三十五贬值了，低谷了的是谁呢？

一类就是跳跳乐，单纯为经济自由和老子干的不开心而跳的，不是指那些因为平台不好而跳的。

三五年内跳跳乐是一件让人开心的事，money蹭蹭蹭的上飙，可是按照一年一家的标准，再加上三个月试用期，倒霉一点六个月试用期，加上一两个月熟悉期，真正接触到的东西少得可怜。

另外一种就是没接触核心开发，干了三五年，都是做集成好的平台，人家封装好的接口配合图形化界面，让你装配，美其名曰实施工程师 (opens new window)，其实相当于流水线工人，换一家公司，换一个系统又得重新学。

第三种就是混日子，小功能也能抄一抄，更多时候在写文档，偶尔找找bug，然后准点下班。一旦让他独立负责一个功能模块，就开始写辞职报告，美其名曰，不知道如何下手，经验不够。

抛除以上三类，哪怕笨一点，哪怕专业不对口，都不可能越活越倒退。不是说没见过笨的，像我这么笨的，我还是见过的。也不是没见过不对口的，有的搞医药搞得风生水起，转行的，不知道今年转回去没有。

当然，年龄大了，有家有业了，难免精力跟不上，但大多数老姜还是辣的，总能一针见血的指点你，当然他们也会犯错，多年的思维成型，难免有时候会固化。

补充一下，加在这里，题主想搞什么平台，得加紧时间。一个惨痛的经验，我的朋友当时搞了一个点餐软件，可以预览菜单，然后电话联系商家，正想发扬光大搞一个能在线点单下单的软件，美团就出来了……

一个人搞不过一群人的，有啥好想法，要趁早，你也知道，大企业有个好点子，只要现成的系统拿来改改就能交付了……（参考一下用友）

最后，奉劝那些不是很热爱的，天赋真的差的，还是慎重考虑，毕竟吃苦也得吃的比那些天赋好的，热爱的多一些。都是挣扎活着，有时候想想，没必要折腾自己。当然，成就感也是有的，至少别人就觉得写代码的，搞电脑也是一个高手。

那些傻子哪里知道，正儿八经的人，谁会抱着显示器搞……
