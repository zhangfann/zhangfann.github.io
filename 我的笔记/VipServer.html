# 客户端

终端从客户端获得服务的ip地址

需部署使用一个dns域名, 部署一个web服务, 提供vipserver服务端ip列表, 客户端可以在此dns地址, 获得vipserver服务端ip列表

# 服务端

服务端集群内数据保持一致, 使用Diamond持久配置中心作为存放数据的数据库

# 数据结构

地址列表: {ip, 端口, 权重, 环境信息(机房, 城市)} 每个服务单独是一个文件, 对应一个地址列表

# 系统设计

# 无侵入客户端

DNS Filter, 当终端请求服务时, 通过DNS拦截, 如果有vipserver监控的服务, 就返回服务响应ip

# 本地缓存

vipserver每台server拥有全量数据, 一台下线, 客户端可以请求其他server

客户端获取服务地址后, 写入本地缓存

# 分量检测

将n个域名分配给m个机器检查, 将n个域名与m进行mod运算, 如果与本机下标相同, 则进行检查, 否则不进行检查

# 负载均衡

客户端以轮询方式访问vipserver服务端, 以做到负载均衡权重计算

对所有地址（ip）的权重求和 那么每个地址的权重就把 sum 划分成了一个一的区间Ｄi。 在 [0,sum] 间随机取浮点值，f = random(0, sum)。 查找满足条件的地址 m，使得 m ∈Ｄi 即可。
